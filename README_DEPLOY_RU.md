# Размещение Flask-проекта на Render

Инструкция для бесплатного размещения проекта бронирования автомобилей на Render.

## Важно про данные

В проекте лежит только тестовый Excel-файл:

```text
input/car_bookings.xlsx
```

На бесплатном Render файловая система временная. Если пользователь добавит бронь через сайт, она может пропасть после перезапуска сервиса или нового деплоя. Для демо и MVP это нормально, но для реальной эксплуатации лучше подключить базу данных.

## Загрузка проекта на GitHub

1. Создайте новый репозиторий на GitHub.
2. Откройте PowerShell в папке проекта:

```powershell
cd C:\CODEX\car_booking
```

3. Инициализируйте Git, если репозиторий еще не создан:

```powershell
git init
```

4. Добавьте файлы:

```powershell
git add .
```

5. Создайте коммит:

```powershell
git commit -m "Prepare Flask car booking app for Render"
```

6. Подключите удаленный репозиторий GitHub:

```powershell
git remote add origin https://github.com/USERNAME/REPOSITORY.git
```

7. Отправьте проект на GitHub:

```powershell
git branch -M main
git push -u origin main
```

Замените `USERNAME` и `REPOSITORY` на имя пользователя и название репозитория.

## Подключение Render

1. Откройте [Render](https://render.com/).
2. Войдите через GitHub.
3. Нажмите `New +`.
4. Выберите `Web Service`.
5. Выберите репозиторий с проектом.
6. Укажите настройки сервиса.

## Настройки Render

Рекомендуемые параметры:

```text
Runtime: Python 3
Build Command: pip install -r requirements.txt
Start Command: gunicorn app:app
Instance Type: Free
```

В проекте также есть `Procfile`:

```text
web: gunicorn app:app
```

Render может использовать его для запуска web-процесса.

## Переменные окружения

Обязательные переменные не нужны. Render сам передает переменную `PORT`, а `app.py` использует ее при локальном запуске:

```python
port = int(os.environ.get("PORT", 5000))
```

Можно добавить необязательную переменную:

```text
SECRET_KEY=любая_длинная_случайная_строка
```

Она нужна для Flask flash-сообщений и сессий. Если переменную не добавить, приложение использует локальное тестовое значение.

## Открытие публичной ссылки

После успешного деплоя Render покажет публичный адрес сервиса, например:

```text
https://car-booking-demo.onrender.com
```

Откройте эту ссылку в браузере. Главная страница покажет таблицу тестовых броней.

## Проверка после деплоя

Проверьте:

- открывается главная страница;
- работает фильтр по дате;
- работает фильтр по автомобилю;
- открывается форма новой брони;
- конфликтующая бронь не добавляется;
- свободная бронь добавляется;
- кнопка `Отменить` меняет статус на `cancelled`;
- открывается страница `Календарь`.
